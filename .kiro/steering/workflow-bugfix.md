워크플로우: 버그 수정

목표
- 버그를 재현 가능하게 만든 뒤, 그 실패를 테스트로 고정하고 수정한다
- 수정 후 회귀 테스트가 남아야 한다

단계
1. 재현 정보 수집
   - 재현 단계
   - 기대 결과와 실제 결과
   - 에러 메시지 또는 핵심 로그(앞뒤 20~40줄)
2. 범위 고정
   - 수정 가능한 폴더와 파일 지정
   - 관련 없는 대규모 리팩터 금지
3. 실패를 테스트로 고정
   - 최소 재현 입력으로 실패 테스트를 먼저 추가
   - 현재는 실패하는 상태가 되어야 함
4. 원인 분석
   - 가능한 원인 2~3개 가설
   - 가장 가능성 높은 것부터 확인
5. 수정
   - 가장 작은 변경으로 해결
   - 입력 검증, 예외 처리 포함
6. 검증
   - 새 테스트 포함 전체 테스트 실행
   - 빌드 또는 린트가 있는 프로젝트면 같이 실행
7. 재발 방지 기록
   - 원인 1줄
   - 다시 안 생기게 하는 규칙 1줄

작업 요청을 받을 때 반드시 출력할 것
- 변경한 파일 목록
- 추가한 테스트 이름 또는 파일
- 실패 로그는 요약만
- 남은 리스크 1~3개
