# STT (Speech-to-Text) 시스템 요구사항

## 필수 요구사항

### 인식률

- **미국/영국 영어** 인식률이 높아야 함
- **중동 사람이 말하는 영어** 인식률이 높아야 함
- **인도 사람이 말하는 영어** 인식률이 높아야 함
- 다양한 악센트와 발음을 정확하게 인식해야 함

### 지속 시간

- **2시간 이상** 연속 녹음/인식이 가능해야 함
- 자동으로 멈추거나 타임아웃되지 않아야 함
- 장시간 사용 시에도 안정적으로 동작해야 함

### 비용

- **무료 API**여야 함
- 사용량 제한이 없거나 충분히 높아야 함
- 추가 비용 없이 운영 가능해야 함

### 기술적 제약

- 오프라인 동작 가능 (선호)
- 브라우저 기반 동작
- 모델 다운로드 후 로컬 실행 가능

## 현재 문제점

### Vosk-browser 한계

- **Large 모델 로딩 불가**: 100MB 이상 모델을 메모리에 로드하지 못하는 버그
- **Small 모델 한계**: 중동/인도 영어 악센트 인식률 낮음
- Large 모델이 필요하지만 기술적으로 불가능

### Web Speech API 한계

- **자동 중단**: 일정 시간(보통 1분) 후 브라우저가 자동으로 인식 중단
- **2시간 이상 불가능**: 장시간 연속 사용 불가능
- 재시작 로직으로 우회 가능하나 불안정

## 금지 사항

- Web Speech API 단독 사용 (자동 중단 문제)
- 유료 API 사용 (OpenAI Whisper, Google Cloud STT, Azure Speech 등)
- 인식률이 낮은 Small 모델만 사용
- 2시간 미만으로 제한되는 솔루션

## 대안 검토 필요

현재 요구사항을 모두 만족하는 솔루션을 찾기 어려운 상황:

1. **Vosk Large 모델 문제 해결**
   - vosk-browser 라이브러리 버그 수정 또는 대체
   - 다른 Vosk 구현 방식 검토 (WebAssembly 직접 빌드 등)

2. **다른 오프라인 STT 라이브러리**
   - Whisper.cpp (WebAssembly 버전)
   - Silero Models
   - Coqui STT

3. **하이브리드 접근**
   - 짧은 세션: Web Speech API (자동 재시작)
   - 긴 세션: 다른 솔루션

## 참고

- 현재 시스템은 Vosk-browser 기반으로 구현되어 있음
- Small 모델은 정상 동작하나 악센트 인식률 부족
- Large 모델은 다운로드는 되나 메모리 로딩 실패 (0%에서 멈춤)
