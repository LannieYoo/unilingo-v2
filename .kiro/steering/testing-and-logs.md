테스트와 로그 규칙

핵심 규칙
1. 테스트 실행 자체는 비용이 아니다
   - 토큰은 대화에 출력되는 텍스트 양에서 늘어난다
2. 로그는 최소로 공유한다
   - 실패한 테스트 이름
   - 첫 에러 메시지
   - 스택트레이스 상단 30줄
   - 필요하면 에러 주변 20~40줄만 추가
3. 실패는 테스트로 고정한다
   - 버그를 재현하는 테스트를 먼저 만든다
   - 수정 후 그 테스트가 통과해야 한다
   - 테스트는 남겨서 회귀를 막는다

권장 커맨드
프론트
- npm test 또는 pnpm test
- npm run build 또는 pnpm build
- npm run lint 또는 pnpm lint

백엔드
- pytest
- ruff 또는 flake8, mypy가 있으면 함께

출력 제한 규칙
- 전체 로그를 그대로 붙이지 않는다
- 요약 템플릿
  1) failing test: 이름
  2) error: 첫 줄
  3) location: 파일:라인
  4) hypothesis: 원인 1~2개
  5) fix: 변경 요약 3줄
