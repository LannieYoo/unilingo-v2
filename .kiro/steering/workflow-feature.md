워크플로우: 기능 추가

목표
- 기존 패턴에 맞게 기능을 추가하고, 최소 테스트를 남긴다

단계
1. 요구사항 정리
   - 해야 하는 것
   - 하지 말아야 하는 것
   - 완료 조건 2~5개
2. 기존 구현 찾기
   - 비슷한 기능의 파일 1~2개를 기준으로 삼기
3. 설계
   - 데이터 흐름: 입력, 처리, 출력
   - 인터페이스: 함수 시그니처, API 스펙, 컴포넌트 props
4. 뼈대 생성
   - 폴더, 파일, 라우트, 컴포넌트 스캐폴딩
5. 구현
   - 작은 단위로 나눠서 구현
   - 경계 조건 처리
6. 최소 테스트 추가
   - 핵심 로직 유닛 테스트 1개 이상
   - 또는 API 통합 테스트 1개 이상
7. 검증
   - 테스트, 빌드, 린트 실행
8. 정리
   - 중복 제거, 타입 정리, 간단 문서 또는 주석

작업 요청을 받을 때 반드시 출력할 것
- 완료 조건 체크 결과
- 변경 파일 목록
- 실행한 검증 커맨드 요약
