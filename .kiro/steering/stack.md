프로젝트 스택 기본 규칙

기본 전제
- 프론트엔드: React + Vite + TypeScript + Tailwind
- 백엔드: FastAPI + Python
- DB: SQL Server 가능
- 패키지 매니저: 프론트 npm 또는 pnpm 중 프로젝트에 이미 쓰는 것 유지
- 테스트 러너: 프론트 Vitest, 백엔드 pytest를 우선 고려
- 운영/배포: 프로젝트에 있는 기존 방식(CI, Docker, scripts)을 우선 따른다

선택 규칙
1. 새 라이브러리 추가는 최소화
2. 기존 코드의 패턴을 먼저 찾고, 같은 방식으로 구현
3. 타입, 입력 검증, 에러 처리의 일관성 유지
4. 성능 최적화는 요구가 있을 때만 한다

출력 형식
- 계획: 체크리스트 5~10개
- 변경 파일: 파일 경로 목록
- 검증: 실행한 커맨드와 결과 요약
