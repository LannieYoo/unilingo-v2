코드 스타일과 안전장치

안전장치
1. 변경 범위를 명확히
   - 수정 가능 폴더를 먼저 선언
2. 작은 변경으로 진행
   - 기능 단위로 쪼개서 커밋 가능한 크기로
3. 기존 패턴 우선
   - 유사 코드 1개를 기준으로 동일한 스타일 적용
4. 타입과 입력 검증
   - TypeScript: any 남발 금지, 가능한 타입 좁히기
   - Python: pydantic 모델, 타입 힌트 유지
5. 에러 처리
   - 사용자 입력 에러와 서버 에러를 구분
   - 의미 있는 에러 메시지와 로그

리뷰 체크리스트
- 요구사항을 충족했는가
- 테스트가 추가되었는가
- 기존 기능을 깨지 않았는가
- 불필요한 파일 변경이 없는가
- 에러 처리와 경계 조건이 있는가

출력 형식
- 변경 요약 5줄 이내
- 변경 파일 목록
- 테스트 추가 여부
