# Step 1: 요구사항 분석

## 단계 정보
- **단계**: 1/10 - 요구사항 분석
- **스킬**: `dev-product_manager`
- **산출물**: `docs/requirements.md`

---

## 실행 단계

### 1. 스킬 로드

`dev-product_manager` 스킬을 로드하여 요구사항 분석 전문 지식과 템플릿을 가져옵니다.

### 2. 배경 정보 수집

사용자와의 대화를 통해 다음 정보를 수집합니다:

**필수 항목**:
- [ ] 프로젝트 이름
- [ ] 프로젝트 배경/동기
- [ ] 대상 사용자 그룹
- [ ] 핵심 문제/페인 포인트

**선택 항목**:
- [ ] 경쟁사 분석
- [ ] 예산/시간 제한
- [ ] 기술 선호도

### 3. 요구사항 분류

수집된 정보를 다음과 같이 정리합니다:

1. **기능 요구사항** (Functional Requirements)
   - 핵심 기능
   - 보조 기능
   - 향후 확장 계획

2. **비기능 요구사항** (Non-Functional Requirements)
   - 성능 요구사항
   - 보안 요구사항
   - 가용성 요구사항
   - 유지보수성 요구사항

3. **제약 조건** (Constraints)
   - 기술적 제약
   - 비즈니스 제약
   - 시간적 제약

### 4. 문서 생성

`docs/requirements.md`를 생성합니다:

```markdown
# {프로젝트 이름} - 요구사항 정의서

## 1. 프로젝트 개요
### 1.1 배경
### 1.2 목표
### 1.3 범위

## 2. 대상 사용자
### 2.1 사용자 페르소나
### 2.2 사용자 시나리오

## 3. 기능 요구사항
### 3.1 핵심 기능
### 3.2 보조 기능

## 4. 비기능 요구사항
### 4.1 성능
### 4.2 보안
### 4.3 가용성

## 5. 제약 조건

## 6. 수락 기준

## 7. 부록
```

### 5. 사용자 확인

생성된 요구사항 정의서를 사용자에게 보여주고 확인을 요청합니다:

```
[C] 확인 - 요구사항 완벽, 다음 단계로 진행
[E] 편집 - 요구사항 수정
[A] 추가 - 요구사항 보충
```
