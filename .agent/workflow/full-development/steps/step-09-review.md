# Step 9: 코드 리뷰

## 단계 정보
- **단계**: 9/10 - 코드 리뷰
- **스킬**: `dev-code_reviewer`
- **입력**: `src/`, `tests/`
- **산출물**: `docs/review-report.md`

---

## 실행 단계

### 1. 컨텍스트 로드

다음 파일을 읽고 분석합니다:
- `docs/architecture.md` - 아키텍처 규격
- `docs/stories.md` - 기능 요구사항
- `src/` - 모든 소스 코드
- `tests/` - 테스트 코드

### 2. 스킬 로드

`dev-code_reviewer` 스킬을 로드하여 코드 리뷰 전문 지식을 활용합니다.

### 3. 리뷰 차원 (Dimensions)

| 차원 | 가중치 | 설명 |
|------|------|------|
| **기능의 정확성** | 30% | 요구사항 충족 여부 |
| **코드 품질** | 25% | 가독성, 유지보수성 |
| **보안성** | 20% | 보안 취약점 확인 |
| **성능** | 15% | 성능 문제 확인 |
| **테스트 커버리지** | 10% | 테스트의 충분성 |

### 4. 자동화 확인

자동화 도구를 실행합니다:

```bash
# 코드 스타일
npm run lint
python -m flake8 src/backend

# 타입 확인
npm run type-check
python -m mypy src/backend

# 보안 스캐닝
npm audit
pip-audit
bandit -r src/backend

# 복잡도 확인
npm run complexity
radon cc src/backend -a

# 의존성 확인
npm outdated
pip list --outdated
```

### 5. 보안 리뷰

#### 5.1 OWASP Top 10 확인
- [ ] **인젝션 (Injection) 공격** - SQL/NoSQL/명령 인젝션
- [ ] **인증 결함** - 취약한 비밀번호, 세션 관리
- [ ] **민감 데이터 노출** - 암호화, 전송 보안
- [ ] **XML 외부 개체 (XXE)** - XML 엔티티 보안
- [ ] **접근 제어** - 권한 확인
- [ ] **보안 설정 오류** - 기본 설정 및 잘못된 설정
- [ ] **크로스 사이트 스크립팅 (XSS)** - 스크립트 인젝션
- [ ] **불안전한 역직렬화 (Deserialization)** - 역직렬화 취약점
- [ ] **알려진 취약점 컴포넌트** - 업데이트되지 않은 라이브러리
- [ ] **부족한 로그 및 모니터링** - 로그 기록 및 추적

#### 5.2 민감 정보 확인
```bash
# 하드코딩된 키 확인
grep -r "password\|secret\|api_key" src/ --include="*.py" --include="*.ts"

# .env 파일 확인
test -f .env && echo "경고: .env 파일이 존재합니다. .gitignore에 포함되었는지 확인하세요."
```

### 6. 성능 리뷰

#### 6.1 백엔드 성능
- [ ] N+1 쿼리 문제
- [ ] 데이터베이스 인덱스 누락
- [ ] 대량의 메모리 할당
- [ ] 블로킹 (Blocking) 작업
- [ ] 캐시 누락

#### 6.2 프론트엔드 성능
- [ ] 대형 번들 (Bundle) 파일
- [ ] 최적화되지 않은 이미지
- [ ] 지연 로딩 (Lazy Loading) 누락
- [ ] 불필요한 리렌더링
- [ ] 메모리 누수

### 7. 코드 품질 리뷰

#### 7.1 명명 규칙
```
✓ 변수명의 의미가 명확함
✓ 함수명이 동작을 설명함
✓ 클래스명에 명사 사용
✓ 상수에 대문자 사용
```

#### 7.2 코드 구조
```
✓ 함수 길이 < 50줄
✓ 클래스 길이 < 300줄
✓ 파일 길이 < 500줄
✓ 루프 중첩 < 3단계
✓ 순환 복잡도 (Cyclomatic Complexity) < 10
```

#### 7.3 베스트 프랙티스
```
✓ DRY - 코드 중복 없음
✓ SOLID 원칙 준수
✓ 완벽한 에러 처리 로직
✓ 합리적인 로그 기록
✓ 명확한 주석 작성
```

### 8. 이슈 분류

| 심각도 | 설명 | 처리 |
|--------|------|------|
| 🔴 Critical (치명적) | 보안 취약점, 데이터 손실 위험 | 반드시 수정해야 함 |
| 🟠 Major (중요) | 기능 결함, 성능 문제 | 수정 권장 |
| 🟡 Minor (사소함) | 코드 스타일, 사소한 개선 | 수정 제안 |
| 🔵 Info (정보) | 정보 메시지, 문서 제안 | 선택적 수정 |

### 9. 보고서 생성

`docs/review-report.md`를 생성합니다:

```markdown
# 코드 리뷰 보고서

## 개요
- 리뷰 날짜: {날짜}
- 코드 버전: {commit_hash}
- 리뷰 범위: src/, tests/

## 점수
| 차원 | 득점 | 설명 |
|------|------|------|
| 기능의 정확성 | 9/10 | 요구사항 충족 |
| 코드 품질 | 8/10 | 양호함 |
| 보안성 | 9/10 | 심각한 취약점 없음 |
| 성능 | 7/10 | 최적화 여지 있음 |
| 테스트 커버리지 | 8/10 | 커버리지 목표 달성 |
| **총점** | **82/100** | **통과** |

## 발견된 이슈

### 🔴 Critical (0)
없음

### 🟠 Major (2)
1. **[BE-001] N+1 쿼리 문제**
   - 파일: `src/backend/services/order_service.py:45`
   - 설명: 루프 내에서 사용자 정보 조회
   - 제안: joinedload를 사용하여 즉시 로딩 처리

2. **[FE-001] XSS 위험**
   - 파일: `src/frontend/components/Comment.tsx:23`
   - 설명: dangerouslySetInnerHTML 사용
   - 제안: DOMPurify를 사용하여 세니타이징 처리

### 🟡 Minor (5)
1. ...

### 🔵 Info (3)
1. ...

## 수정 제안

### 필수 수정 (배포 차단)
- [ ] N+1 쿼리 문제 해결
- [ ] XSS 위험 해결

### 수정 권장 (다음 버전)
- [ ] 이미지 로딩 최적화
- [ ] 캐시 로직 추가

## 결론

코드의 전반적인 품질은 양호하며, 2개의 Major 이슈를 수정한 후 배포할 것을 권장합니다.
```

### 10. 이슈 수정

Critical 및 Major 이슈에 대해 다음 과정을 수행합니다:

```
┌─────────────────────────────────────────────┐
│  Issue: [BE-001] N+1 쿼리 문제               │
├─────────────────────────────────────────────┤
│  1. 문제가 발생한 코드 위치 확인              │
│  2. 수정 사항 적용                           │
│  3. 테스트 추가/업데이트                     │
│  4. 재리뷰 실행                              │
│  5. 수정 완료 표시                           │
└─────────────────────────────────────────────┘
```

### 11. 사용자 확인

```
리뷰 요약:
- 🔴 Critical: 0
- 🟠 Major: 2 (수정 완료)
- 🟡 Minor: 5
- 🔵 Info: 3

[C] 확인 - 리뷰 통과, 다음 단계로 진행
[F] 수정 - 이슈 수정 계속
[R] 재리뷰 - 다시 리뷰 실행
```
